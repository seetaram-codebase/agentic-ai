When you look at a vector in Pinecone's web interface, you see something like:


**This makes it LOOK like only text is stored - but that's just the UI showing you the metadata!**

## Where Are the Embeddings?

### The `values` Field = THE EMBEDDINGS! ✅

You showed me earlier:

```
values: [-0.0701156333, -0.0192103069, 0.0267699808, ...]
```

**These ARE the embeddings!** This 1536-number array is the vector embedding generated by Azure OpenAI.

### Why It's Not More Visible in Pinecone UI

The Pinecone web console **minimizes the display of the `values` field** because:
1. It's 1536 numbers - too long to display fully
2. Humans can't read/interpret these numbers anyway
3. The metadata (especially `text`) is more useful for humans to verify what's stored

### Method 3: Query via Pinecone API

```python
import os

# Fetch one vector by ID
```

## Why Both Embeddings AND Text Are Needed

### The Embeddings (values):
- **Purpose**: Enable semantic search
- **Used by**: Vector similarity algorithms
- **Not human-readable**: Just numbers like [-0.07, 0.02, ...]

### The Original Text (metadata.text):
- **Purpose**: Show results to users and LLM
- **Used by**: Application to display context
- **Human-readable**: The actual document content

### The Search Process:

```
1. User Query: "What links are in the document?"
   ↓
2. Convert to embedding: [0.05, -0.12, 0.08, ...]
   ↓
3. Pinecone searches EMBEDDINGS (values field):
   - Compares [0.05, -0.12, ...] against all stored vectors
   - Finds most similar: [−0.07, -0.01, 0.02, ...] (Score: 0.89)
   ↓
4. Return METADATA.TEXT from matching vectors:
   - "https://app.nuclino.com/..."
   ↓
5. Send to Azure OpenAI as context:
   - Context: "https://app.nuclino.com/..."
   - Question: "What links are in the document?"
   - Answer: "The document contains a link to..."
```

## Why the UI Focuses on Text

The Pinecone web interface shows metadata (text) prominently because:

1. **Developers need to verify** the right content is stored
2. **Numbers are meaningless** to humans - can't verify correctness
3. **Text is searchable** - you can browse/debug what's in the index

But the embeddings are ALWAYS there in the `values` field!

## Summary

✅ **Embeddings ARE stored** - in the `values` field (you showed me 1536 numbers)
✅ **Text is ALSO stored** - in `metadata.text` field for display
✅ **Both are required** for RAG to work:
   - Embeddings for similarity search
   - Text for context/display

The Pinecone UI just **minimizes** the display of the values array because it's not human-readable!
